Buenas Practicas de edicion de codigo en JS

Cuando hablamos de buenas prácticas en programación, estamos tomando en cuenta metodologías, técnicas y principios que se tiene que implementar a la hora de crear un software, esto es para que se vuelva de fácil entendimiento, mejore su velocidad y sea más seguro de mantener y desplegar. A Continuación habláremos de algunas buenas Prácticas a la hora de crear código JS.

  1. 1. Utiliza siempre Punto y coma. Si no colocas punto y coma, normalmente, los navegadores no tendrán problemas con estas omisiones, pero, se trata de una muy mala práctica que puede ser creadora de problemas difíciles de encontrar en códigos de gran tamaño.

  2. 2. Comenta tu código. Al inicio puede parecer al innecesario, pero necesitas comentar tu código de la mejor manera posible, por la sencilla razón es que meses después de la creación de algún bloque del código no recuerdes como fue tu línea de pensamiento en ese momento, o si alguna otra persona necesita entender lo que escribiste en ese bloque de código, por eso comenta la secciones importantes del código que estés creando.

  3. 3. Omite la palabra var. Cuando quieras crear muchas variables, recuerda que las puedes sustituir por una “,”, esto ayuda a que el código sea un más legible y limpio.

  4. 4. Utiliza varios parámetros. Utiliza varios parámetros de entrada, ya que estoy ayuda a un mejor entendimiento del código, además ayuda al rendimiento de la aplicación, porque ya no necesita crear parámetros de objetos.

  5. 5. Utiliza las funciones de flechas. Es una forma de concisa de escribir funciones en JavaScript y es una forma de mantener tu código limpio, también resuelve problemas a la hora de acceder a las devoluciones de llamada de las propiedades.

  6. 6. Declarar variables afuera de For. Cuando ejecutes sentencias del tipo for no es necesario que se trabaje de más, ya que cada vez que se repite la iteración hará todos los procesos dentro de ella. Ejemplo de buena práctica con for.

  7. 7. Operador spread. El operador (…) es capaz de expandir arrays con una sola línea de código, este operador se utiliza al querer combinar parámetros. Esto nos ayuda también a tener un código limpio y fácil de entender.

  8. 8. Colocar los scripts en la parte inferior del html. El objetivo para una página es que cargue lo más rápidamente posible, por eso tenemos que facilitar esto para el navegador, al colocar los archivos que solo aportan funcionalidad en al página se pueden colocar en la parte inferior, antes del cierro del cuerpo del archivo.

  9. 9. No usar la vía corta. Técnicamente las llaves se pueden omitir en la mayoría de los navegadores. Solo se recomienda no utilizarlas en líneas simples. Aun que es debatible su utilización o no en este caso.



¿Que es Linting y para que se utiliza en JS?

Al momento de redactar código el surgimiento de errores es inevitable, algunos de estos errores causar problemas en la interfaz de la aplicación incomodando al usuario y otros afectan al funcionamiento del mismo. Existe un cierto grupo de errores que pueden identificarse y corregirse antes de la ejecución del código, entre estos se encuentran:

  • Errores de sintaxis
  • Código poco intuitivo o difícil de mantener.
  • Uso de “malas prácticas”.
  • Uso de código inconsistente.

El grupo de errores mencionados con anterioridad son mas comunes a diferencia de otros más graves debido a que son los menos evidentes.

El termino linting se designa a una herramienta que “revisa” el código en busca de fallos o errores que afecten al mismo. Algunos “linteres” pueden brindar sugerencias de como solucionar el error o incluso como esta herramienta puede solucionarlos.

Las herramientas de linting pertenecen a un grupo de programas conocidos como herramientas de análisis estático un proceso de revisión de un programa sin ejecutar dicho programa, por lo general la revisión se realiza sobre el código fuente o alguna clase de código objeto.

Un ejemplo de esta herramienta es ESLint.

¿Qué es ESLint?

ESLint es una herramienta de código abierto enfocada en el proceso de "lintig" para JavaScript. ESLint es la herramienta predominante para la tarea de "limpiar" código JavaScript tanto en el servidor (node.js) como en el navegador.

Dado que javascript es un lenguaje dinámico y de tipado débil, es especialmente fácil caer en errores humanos a la hora de escribir código. ESLint utiliza un sistema de reglas que permiten definir qué es y que no es posible dentro del código. ESLint está escrito en Nodejs y es posible instalarlo desde npm.



Funciones Flecha

Estas funciones mas compactas a comparación de las funciones tradicionales ya que estas permiten escribir método más concisos o simplificados, aprovechando las características de retorno implícito y el no uso de paréntesis, sin embargo, su uso es mas limitado y no se pueden utilizar en cualquier situación a comparación de las funciones tradicionales, no cuentan con su propio enlace a this o super y no debe usarse como método, a comparación de las funciones tradicionales estas no cuentan con un argumento o palabra clave y estas generalmente se basan en establecer un ámbito o enlace, estas no pueden utilizarse como método constructor.

Estas al ser una sintaxis abreviada, nos ahorramos mas tiempo al momento de desarrollo el código, además de simplificar el ámbito de la función, es una de las características más utilizadas de ES6 (se encarga de regir como debe ser interpretado y funcionar el lenguaje JavaScript).
